## 2. 내가 만든 DCO 인프라 용어 정의

### Rack
- **내 정의:** 수많은 서버와 네트워크 장비들을 서랍장처럼 차곡차곡 위로 쌓아 올려 안전하게 보관하는 표준 규격의 금속 캐비닛입니다. `SAMPLE-RACK-01`처럼 장비의 물리적 위치를 식별하는 기본 단위가 됩니다.

### Server
- **내 정의:** 인터넷을 통해 전 세계 사용자들이 홈페이지를 보거나 데이터를 저장할 수 있도록 24시간 내내 서비스를 제공하는 고성능 컴퓨터입니다. 일반 PC와 달리 랙에 가로로 얇게 장착되는 형태가 특징입니다.

### NIC (Network Interface Card)
- **내 정의:** 컴퓨터나 서버를 대형 랜선이나 광케이블에 연결하여 외부와 데이터를 주고받을 수 있게 해주는 물리적인 통신 카드 부품입니다. 부품이나 케이블 접촉에 문제가 생기면 서버의 통신이 완전히 차단될 수 있습니다.

### ToR Switch (Top of Rack Switch)
- **내 정의:** 하나의 랙 맨 위(Top)에 설치되어, 그 랙 안에 들어있는 수십 대의 서버들을 하나로 묶어 외부 네트워크망으로 연결해 주는 대형 분배 장치입니다. 이 장비에 문제가 생기면 해당 랙 전체 서버가 먹통이 될 수 있습니다.

### Link Down
- **내 정의:** 서버나 스위치 사이의 케이블 연결이 물리적으로 끊어지거나 신호가 단절되어 통신이 불가능해진 현상입니다. 케이블 불량, 포트 접촉 불량, 혹은 장비 전원 차단 등 다양한 원인이 있을 수 있습니다.

### CRC error (Cyclic Redundancy Check error)
- **내 정의:** 데이터가 케이블선을 타고 이동하는 과정에서, 미세한 먼지나 신호 간섭 때문에 데이터 일부가 깨지거나 변형되어 전달된 오류입니다. 완전히 끊어진 것은 아니나, 케이블 노후화나 커넥터 오염이 의심되는 신호입니다.

### SSD failure (Solid State Drive failure)
- **내 정의:** 데이터를 저장하는 장치인 SSD의 수명이 다해가거나 내부에 치명적인 결함이 발생하여 교체가 필요함을 알리는 경고입니다. 데이터 유실을 막기 위해 핫스왑(무중단 교체) 가이드라인을 따라야 합니다.

### Fan Alert
- **내 정의:** 서버 내부의 뜨거운 열기를 뿜어내 주는 냉각 팬이 제대로 돌지 않거나 속도가 떨어졌을 때 발생하는 센서 경고입니다. 이를 방치하면 장비 과열로 인한 시스템 강제 셧다운이 발생할 수 있습니다.

### SOP (Standard Operating Procedure)
- **내 정의:** 현장의 복잡하고 위험한 수반 작업을 누구나 오차 없이 완수할 수 있도록 단계별 행동 규칙을 정리한 표준 작업 절차서입니다. 실습자가 임의로 장비를 만지지 않게 제어하는 기준이 됩니다.

### SLA (Service Level Agreement)
- **내 정의:** 인프라 서비스의 가동률이나 긴급 장애 대응 속도 등을 명시하여 고객과 체결하는 공식적인 품질 보증 계약입니다. 인시던트 발생 시 업무 카드의 우선순위를 정하는 척도가 됩니다.

### Escalation
- **내 정의:** 현장 근무자가 단독으로 해결할 수 없거나 권한 범위 밖의 문제가 생겼을 때, 상위 담당 엔지니어나 책임 부서에 신속히 공유하여 지원을 요청하는 보고 절차입니다.

### Incident Report
- **내 정의:** 예기치 못한 하드웨어 장애나 중단 이벤트가 발생했을 때 시점별 전개, 원인 규명, 조치 결과 및 재발 방지 대책을 투명하게 문서화하는 대응 보고서입니다.

### Handover Memo
- **내 정의:** 교대 근무 시 이전 근무자가 진행 중이던 작업 내역이나 유의 깊게 관찰해야 할 하드웨어 특이사항을 다음 근무 조에 공백 없이 공유하기 위한 인수인계 문서입니다.

### Temperature warning
- **내 정의:** 서버실 내부나 특정 랙의 온도가 임계치 이상으로 상승하여 과열 조짐이 보일 때 시스템이 자동 발생시키는 경고 신호입니다. 공조 장치 점검 및 통풍로 확인 단계로 이어집니다.

## 3. 헷갈리는 용어 비교

- **내가 비교한 용어:** Link Down vs CRC error
- **내가 이해한 차이:** `Link Down`은 케이블이 뽑히거나 전원이 꺼져 통신 신호가 완전히 끊어진 상태인 반면, `CRC error`는 연결은 되어 있으나 먼지나 선로 노후화로 인해 데이터가 깨져서 불완전하게 전송되는 상태를 의미합니다.
- **아직 헷갈리는 부분:** 소프트웨어 설정 오류 때문에 `Link Down`이나 `CRC error`가 발생하기도 하는지, 그 구분 기준이 아직 조금 헷갈립니다.

## 4. 앞으로 사용할 학습 보조 프롬프트

### 프롬프트 1. 모르는 DCO 용어 설명용
- **역할:** DCO 인프라 직무를 처음 배우는 학생을 돕는 학습 도우미
- **목표:** 입력된 DCO 관련 전문 용어를 비전공자 대학생도 상식선에서 파악할 수 있게 1~2문장으로 쉽게 설명해 주세요.
- **상황:** 공개 직무 맥락과 교육용 예시(`SAMPLE`, `DEMO` 등)를 기준으로 설명하며, 학생이 직접 자기 언어로 바꿀 수 있도록 안내하는 상황입니다.
- **출력 형식:** 1. 쉬운 설명, 2. 활용 맥락, 3. 주의할 점 구조로 요약해 주세요.
- **제한 조건:** 실제 회사 내부 절차를 단정하거나, 실제 티켓 번호, 장비 시리얼 번호, 실제 IP 주소는 포함하지 마세요.

### 프롬프트 2. 비슷한 용어 비교용
- **역할:** DCO 인프라 용어를 명확하게 비교해 주는 프롬프트 코치
- **목표:** 입력된 비슷해 보이는 두 DCO 용어의 개념적, 현장 업무적 차이점을 명확히 구분해 주세요.
- **상황:** 학생이 두 용어를 혼동하여 가상 장애 시나리오 분석에 어려움을 겪고 있는 교육적 상황입니다.
- **출력 형식:** 1. 용어 A/B의 뜻, 2. 가장 큰 차이점, 3. 헷갈리지 않는 기억법 구조로 작성해 주세요.
- **제한 조건:** 특정 원인을 하나로 단정 짓지 마세요. 실제 회사명이나 기밀 정보, 실제 인프라 보안 규칙은 제외하세요.

### 프롬프트 3. 장애 표현 확인 순서 정리용
- **역할:** 데이터센터 인프라 장애 로그 분석을 보조하는 교육용 기술 멘토
- **목표:** 주어진 가상의 에러 로그를 보고, 현장 엔지니어가 취해야 할 논리적인 확인 순서와 점검 체크리스트 초안을 정리해 주세요.
- **상황:** 교육용 샘플 상황(`SAMPLE-RACK-01` 등)에서 원인을 성급히 단정하지 않고 절차대로 확인하는 태도를 기르는 수업 중입니다.
- **입력:** [가상의 에러 로그 입력 예: Link Down / Fan Alert 등]
- **출력 형식:** 1. 예상되는 다각적 원인, 2. 단계별 확인 순서(SOP 예시), 3. 상위 보고(Escalation) 시 취합할 데이터
- **제한 조건:** 실제 장비 시리얼이나 내부 정보를 유추할 수 있는 표현은 절대 금지하며, IP 주소 필요 시 `192.0.2.x`와 같은 문서용 주소만 사용하세요.

## 5. AI 답변을 보고 내가 수정한 점

- **어려워서 쉽게 바꾼 표현:** "네트워크 인터페이스 카드(NIC)의 패킷 드롭 현상" -> "연결선 접촉 불량으로 인한 통신 차단 현상"으로 직관적이게 변경
- **실제 정보처럼 보일 수 있어 삭제하거나 수정한 표현:** AI 답변 중 임의로 들어간 장비 모델명 및 가상의 티켓 번호(`#TKT-1023`)를 발견하여 `SAMPLE-RACK`, `교육용 시나리오`로 전면 교체
- **너무 단정적이라 바꾼 표현:** "CRC 에러는 무조건 광케이블 먼지 때문이다" -> "광케이블 커넥터 오염이나 선로 노후화 등 다양한 원인이 있을 수 있다"로 다각화
- **아직 헷갈리는 용어:** ToR Switch와 일반 Core Switch의 물리적 배치 구성도
- **다시 공부하고 싶은 용어:** 무중단 부품 교체 기술인 `Hot-Swapping`과 `SSD failure` 발생 시의 안전 탈착 절차

## 6. 오늘의 느낀 점

- **오늘 처음 알게 된 용어는:** ToR(Top of Rack) 스위치 (랙의 맨 위에서 허브 역할을 해주는 대형 장치가 따로 있다는 것을 처음 알게 됨)
- **가장 헷갈렸던 용어는:** 끊어진 것(`Link Down`)과 깨진 것(`CRC error`)의 현장 발생 메커니즘 구분
- **AI 답변을 사용할 때 조심해야 할 점은:** AI가 제시하는 기술 설명 중 실제 기업 내부의 대외비 절차처럼 단정 짓는 과장 표현을 걸러내야 하며, 하드웨어 실물 시리얼이나 비공개 명칭은 보안상 프롬프트에 절대 쓰면 안 됨을 명심함
