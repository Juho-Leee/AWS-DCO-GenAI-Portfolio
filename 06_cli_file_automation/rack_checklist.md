## 교육용 랙 확인 체크리스트 (EDU_SAMPLE_RACK_CHECKLIST)

본 체크리스트는 제공된 샘플 로그와 샘플 티켓 데이터를 읽고 분석 결과를 기록하기 위한 가이드라인입니다.

### [체크리스트 항목]

1.  **발생 시간 확인 (Verify Event Time)**
    *   [ ] 샘플 티켓 및 로그 상에 기록된 이벤트 발생 시간이 일치하는지 확인 및 기록:
        *   *기록 란 (예: 2026-07-07 15:10:00)*: \_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_
2.  **샘플 장비명 확인 (Verify Sample Device Name)**
    *   [ ] 오류가 감지된 샘플 장비의 이름을 기록 (반드시 `SAMPLE_` 또는 `EDU_` 접두어 적용):
        *   *기록 란 (예: SAMPLE_TOR_SW_01)*: \_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_
3.  **반복 여부 확인 (Check Recurrence)**
    *   [ ] 과거 샘플 이력 또는 로그 중 동일한 샘플 장비(`SAMPLE_TOR_SW_01` 등)에서 유사 이벤트가 발생했는지 확인:
        *   *기록 란 (단발성 / 주기적 반복 / 확인 불가)*: \_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_
4.  **Ticket 상태 확인 (Check Ticket Status)**
    *   [ ] 분석 대상 티켓이 신규(`EDU-NEW`), 처리 중(`EDU-IN-PROGRESS`), 완료(`EDU-RESOLVED`) 중 어떤 상태인지 확인:
        *   *기록 란*: \_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_
5.  **원인 단정 금지 (Do Not Conclude Cause Prematurely)**
    *   [ ] 단순 에러 카운트 증가만으로 하드웨어 결함이나 케이블 불량 등으로 원인을 섣불리 단정하지 않았는지 점검:
        *   *점검 여부 (확인 완료)*: [ ]
6.  **보고서 초안 작성 메모 (Memo for Drafting Incident Report)**
    *   [ ] 향후 보고서 작성을 위해 관찰 내용 및 조치 내역의 요약본을 기록할 수 있는 가상의 메모 공간:
        *   *메모 내용*: \_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_

### 보안 가이드 (Security Guide)
*   **주의**: 가상의 케이블 교체나 물리 핀 점검 등 외부 대상에 실질적인 점검 명령을 실행하거나 제안하지 마십시오.
*   **주의**: 체크리스트 작성 시 실제 IP 주소, 장비의 고유 시리얼 넘버, 실서버 티켓 번호, 개인/고객 정보가 포함되지 않도록 하십시오.
